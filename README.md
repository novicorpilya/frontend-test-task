# 🛒 React Carts Management App

Single Page Application (SPA) для управления списком корзин пользователей, написанное на React + Vite с использованием современных архитектурных паттернов. Проект реализован в рамках тестового задания.

---

## 🚀 Инструкция по запуску

Для запуска проекта локально вам потребуется установленный [Node.js](https://nodejs.org/) (рекомендуется версия 18+).

1. **Клонируйте репозиторий:**
   ```bash
   git clone <url-вашего-репозитория>
   cd frontend-test-task
   ```

2. **Установите зависимости:**
   ```bash
   npm install
   ```

3. **Запустите development-сервер:**
   ```bash
   npm run dev
   ```
   > Приложение будет доступно по адресу `http://localhost:5173`

4. **Сборка для Production:**
   ```bash
   npm run build
   ```

## 🏗 Архитектура и Технологический стек

Проект спроектирован с упором на производительность, строгую типизацию и переиспользуемость.

### Core-технологии
*   **React (на базе Vite)** — обеспечивает молниеносную скорость HMR при разработке и оптимизированный бандл в production.
*   **TypeScript** (`strict: true`) — используется повсеместно для описания бизнес-сущностей (`Cart`, `Product`), API-ответов и пропсов компонентов, исключая runtime ошибки.

### Работа с данными (Data Layer)
*   **TanStack React Query (v5)** — мощный инструмент для работы с серверным состоянием (Server State).
    *   Решает проблему кэширования: при возврате со страницы просмотра заказа к списку, данные берутся из кэша мгновенно.
    *   Обработка `isLoading` и `isError` инкапсулирована "из коробки".
    *   Мутации (`useMutation`) мгновенно зашивают результат API в локальный кэш через `queryClient.setQueryData`, что избавляет UI от "миганий" и загрузок ("Optimistic / Manual UI Update").
*   **Axios** — HTTP-клиент, вынесенный в отдельный инстанс (`api/client.ts`) для возможности легко внедрять перехватчики (interceptors).

### Клиентское состояние (Client State)
*   **Zustand** — легковесный менеджер состояний. Сознательно выбран вместо Redux за счет отсутствия бойлерплейта. Хранит глобальное UI-состояние: `limit` и `skip` (пагинацию). Это позволяет юзеру "провалиться" в детали 3-ей страницы и при нажатии "Назад" вернуться строго на нее.

### UI / UX слой
*   **@emotion/styled** — подход CSS-in-JS. Стили компонентов инкапсулированы, отсутствует риск пересечения классов, доступна мощная темизация.
*   **Skeleton Loading** — во время загрузки пользователь не видит пустой экран, а наблюдает анимированный каркас (`SkeletonBox`).
*   **React Error Boundaries** — приложение обернуто в глобальный перехватчик ошибок (`ErrorBoundary`). Если в компоненте произойдет краш, он не "положит" всё приложение (белый экран), а покажет красивый fallback-UI.

### Структура директорий (Feature-Sliced Design элементы)
```text
src/
├── api/          # Слой слоя взаимодействия с сервером (axios, endpoints API)
├── components/   # Переиспользуемые UI-компоненты (Table, Skeleton, Pagination)
├── hooks/        # Бизнес-логика компонентов и интеграция React Query
├── pages/        # Компоненты-экраны (CartsPage, CartDetailPage)
├── store/        # Глобальные стейты Zustand
└── types/        # Интерфейсы TypeScript
```

## 🧩 Требования ТЗ (Чек-лист)

✅ **Страница списка корзин**
- Данные рендерятся в адаптивной таблице (`overflow-x: auto`).
- Реализована пагинация со стейтом в Zustand (сохранность при переходах).
- Состояния загрузки реализованы через CSS-анимацию Skeleton.

✅ **Детальная страница корзины**
- Динамический роутинг `/carts/:id`.
- Расчет и показ сумм, скидок с перечеркнутыми старыми ценами.

✅ **Редактирование корзины**
- При изменении количества (+/-) или удалении (🗑), отправляется PUT запрос к DummyJSON (`merge: false`).
- UI обновляется мгновенно через ручную мутацию кэша React Query, включая суммы как на детальной странице, так и на странице списка!
- Реализована блокировка UI (disabled состояния кнопок) в момент "полета" запроса.
